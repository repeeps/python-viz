PART B · 별칭과 복사 — CH 04

재바인딩 vs 제자리 변경

파이썬에서 "바꾼다"엔 완전히 다른 두 가지가 있어요. 재바인딩(x = [4,5,6])은 이름표 x를 다른 객체로 옮기는 일 — 원래 객체는 그대로예요. 제자리 변경(x.append(3))은 x가 가리키는 그 객체의 내용을 바꾸는 일 — 객체는 같고(id 유지) 안만 달라지죠. 코드만 보면 둘 다 "x를 바꿨다"처럼 보이지만, 메모리에서 벌어지는 일은 정반대예요. 이 구분이 다음 챕터들 함정의 뿌리예요.

Step 00 — Rebinding Moves the Tag

재바인딩: 이름표를 옮긴다

x = [1,2] 다음 x = [4,5,6] 을 하면, [1,2]가 [4,5,6]으로 변신하는 게 아니에요. 새 리스트 객체가 따로 만들어지고, 이름표 x가 그쪽으로 옮겨 붙는 거죠. 원래 [1,2] 객체는 손대지 않은 채 그대로 있어요(아무도 안 가리키면 청소). 대입(=)은 늘 이렇게 "이름표 옮기기"예요.

→ x = ... 는 항상 재바인딩(이름표 이동). 원래 객체는 안 건드림.

Step 01 — Mutating Changes the Object

제자리 변경: 객체 안을 바꾼다

x.append(3) 이나 x[0] = 9 는 이름표를 옮기지 않아요. x가 가리키는 그 객체의 내용을 직접 바꾸죠. 객체는 같은 자리에 그대로(id 유지), 안의 내용만 달라져요. 가변 객체(리스트·딕트)만 이게 가능해요. 버튼으로 같은 객체 안이 바뀌는 걸 보세요 — 이름표는 한 자리에 가만히 있어요.

→ .append()·[i]=… 는 제자리 변경(같은 객체). 이름표는 안 움직임.

Step 02 — Tell Them Apart by id

id가 바뀌면 재바인딩, 그대로면 제자리

둘을 확실히 구분하는 법: 객체의 id를 보면 돼요. 재바인딩은 이름표가 다른 객체로 가니 id가 바뀌어요. 제자리 변경은 같은 객체를 손보니 id가 그대로죠. 토글로 같은 "결과 [1,2,3]"이 두 방식에서 id가 어떻게 다른지 보세요. 이 차이가 — 같은 객체를 다른 이름도 가리킬 때(별칭) — 사고로 이어져요. 다음 챕터 예고.

→ id 바뀜=재바인딩 / id 유지=제자리 변경. 별칭이 있으면 이 차이가 결정적.

Step 03 — Takeaways

정리

재바인딩(x = …)은 이름표를 새 객체로 옮기고(id 변경, 원래 객체 무사), 제자리 변경(x.append·x[i]=…)은 같은 객체 안을 바꿔요(id 유지). 코드 겉모습은 비슷해도 정반대 일이죠. 그런데 — 같은 객체에 이름표가 여럿 붙어 있을 때(별칭) 한 이름표로 제자리 변경을 하면? 다른 이름표에도 그 변화가 보여요. 그게 파이썬 초보의 가장 흔한 버그예요. 다음 챕터.
