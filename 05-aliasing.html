PART B · 별칭과 복사 — CH 05

별칭 — 한 객체, 여러 이름

y = x 는 리스트를 복사하지 않아요 — 같은 객체에 이름표 y를 하나 더 붙일 뿐이죠(CH01). 그럼 둘은 별칭(같은 객체의 다른 이름)이고, 한쪽으로 제자리 변경하면 다른 쪽에도 그대로 보여요. 이게 파이썬 초보의 1번 버그예요. 그리고 결정적 — 함수에 인자를 넘기는 것도 똑같은 이름표 붙이기라서, 함수 안에서 리스트를 바꾸면 바깥 리스트도 바뀝니다.

Step 00 — y = x Is an Alias

한쪽을 바꾸면 다른 쪽도 바뀐다

x = [1, 2] 다음 y = x — 이제 x와 y는 같은 객체를 가리키는 두 이름표예요(별칭). 여기서 x.append(3) 으로 제자리 변경을 하면? y로 봐도 [1, 2, 3] 이에요 — y를 건드린 적도 없는데! 버튼으로 따라가 보세요. 둘이 같은 객체를 보고 있으니 당연한 일이지만, 처음엔 깜짝 놀라죠.

→ 별칭 = 같은 객체의 여러 이름. 제자리 변경은 모든 이름표에 보임.

Step 01 — Arguments Are Aliases Too

함수에 넘기는 것 = 이름표 붙이기

함수 호출 f(data) 는 마법이 아니에요 — 매개변수 이름(lst)을 인자 객체에 이름표로 붙이는 것, 즉 lst = data 와 똑같아요. 그래서 data 와 lst 는 별칭! 함수 안에서 lst.append(99) 로 제자리 변경하면 — 바깥 data 도 바뀌어 있어요. "분명 안 건드렸는데 리스트가 변했다"는 미스터리의 정체예요.

→ 인자 전달 = 바인딩. 함수 안 제자리 변경은 바깥 객체에 그대로 반영.

Step 02 — Not "By Value" or "By Reference"

"값/참조 전달" 이분법은 잊자

다른 언어에서 온 사람은 "파이썬은 값 전달이야, 참조 전달이야?"를 묻지만 — 둘 다 아니에요. 정답은 "객체에 이름표 붙이기"(바인딩). 그래서 함수 안에서 제자리 변경(lst.append)은 바깥에 보이지만, 재바인딩(lst = [...])은 안 보여요 — 재바인딩은 함수 안 지역 이름표만 옮기거든요(CH04). 토글로 두 경우를 비교하세요. 이게 "어떨 땐 바뀌고 어떨 땐 안 바뀌는" 혼란의 진짜 답이에요.

→ 제자리 변경=바깥에 보임 / 재바인딩=지역 이름표만 이동(바깥 그대로). 값/참조 이분법이 아니라 바인딩.

Step 03 — Takeaways

정리

y = x 와 함수 인자 전달은 둘 다 같은 객체에 이름표 붙이기(별칭). 별칭에 제자리 변경을 하면 모든 이름표에 보이고, 재바인딩은 그 이름표 하나만 옮겨 다른 데엔 안 보여요. "값/참조 전달"이 아니라 "바인딩"으로 보면 전부 일관돼요. 그럼 — 정말로 독립된 사본이 필요하면 어떻게 할까요? "복사하면 되지" 싶지만, 거기에도 함정이 있어요. 다음 챕터.
