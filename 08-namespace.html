PART C · 이름이 사는 곳 — CH 08

네임스페이스

이름표는 허공에 붙지 않아요 — 늘 어떤 방(네임스페이스)에 붙어요. 네임스페이스는 그냥 "이름 → 객체" 매핑이에요. 프로그램 전체가 쓰는 전역 방이 있고, 함수를 호출할 때마다 그 함수만의 지역 방이 새로 생겼다 사라져요. 핵심은 — 방이 다르면 같은 이름이어도 서로 다른 이름이라는 것. 함수 안 x 와 전역 x 는 이름만 같을 뿐 다른 방의 다른 이름표예요.

Step 00 — Names Live in Rooms

네임스페이스 = 이름→객체 방

모듈(파일) 하나를 실행하면 전역 네임스페이스라는 방이 만들어져요. x = 5, name = "kim" 처럼 대입할 때마다 이 방에 "이름 → 객체" 항목이 추가되죠. 네임스페이스는 사실 딕셔너리예요 — 그래서 이름을 넣은 순서도 기억해요. 이름을 쓰면 파이썬은 이 방에서 그 이름표를 찾아 가리키는 객체를 꺼내요.

→ 네임스페이스 = 이름→객체 매핑(방). 전역 방엔 모듈의 이름들이 모여요.

Step 01 — Each Call, a New Room

함수를 부르면 방이 생긴다

함수를 호출하면 그 함수만의 지역 네임스페이스(방)가 새로 만들어져요. 매개변수와 함수 안에서 대입한 이름들이 이 방에 살죠. 함수가 끝나면 그 방은 통째로 사라져요(그래서 지역 변수는 함수 밖에서 안 보임). 버튼으로 호출→방 생성→종료→소멸을 따라가 보세요. 부를 때마다 새 방이에요.

→ 호출마다 지역 방 생성, 종료 시 소멸. 지역 이름은 그 방 안에서만 살아요.

Step 02 — Same Name, Different Rooms

이름이 같아도 방이 다르면 남남

전역 방에 x = 1 이 있고, 함수 안에서도 x = 99 를 쓰면 — 둘은 다른 방의 다른 이름표예요. 함수 안 x 를 바꿔도 전역 x 는 멀쩡하죠. 이름이 겹쳐도 충돌하지 않는 이유예요. 그럼 파이썬은 x 라고 썼을 때 어느 방의 x를 쓸지 어떻게 정할까요? 그게 다음 챕터(LEGB)예요.

→ 방이 다르면 같은 이름도 별개. 어느 방을 쓸지의 규칙 → 다음 챕터.

Step 03 — Takeaways

정리

이름표는 네임스페이스(방)에 살아요 — 모듈의 전역 방, 호출마다 생겼다 사라지는 함수의 지역 방. 방이 다르면 같은 이름도 다른 이름이라 충돌하지 않죠. 그럼 코드에서 x 를 그냥 썼을 때, 파이썬은 여러 방 중 어디서 그 이름을 찾을까요? 정해진 순서가 있어요. 다음 챕터.
