PART B · 별칭과 복사 — CH 07

가변 기본 인자 함정

파이썬에서 가장 유명한 함정. def f(items=[]): 처럼 기본값으로 빈 리스트를 쓰면, 그 리스트는 함수를 부를 때마다 새로 생기는 게 아니에요. def 줄이 실행(정의)될 때 딱 한 번 만들어져, 이후 모든 호출이 같은 하나의 리스트를 공유해요(별칭!). 그래서 호출할수록 내용이 쌓이는 기묘한 일이 벌어지죠. 원인도, 고치는 법도 — 앞에서 배운 별칭·바인딩으로 전부 설명돼요.

Step 00 — The Accumulating Mystery

부를 때마다 쌓인다

def add(v, items=[]): — 매번 빈 리스트로 시작할 것 같죠? add(1) 을 세 번 부르면 [1], [1], [1] 이 나올 거라 기대해요. 그런데 실제론 [1], [1, 1], [1, 1, 1] 로 쌓여요. 버튼으로 직접 호출해 보세요 — 빈 리스트가 매번 새로 안 생기고 이전 결과가 남아 있어요.

→ 기대: 매번 빈 []. 현실: 호출마다 쌓임. 같은 리스트를 재사용하고 있어요.

Step 01 — Evaluated Once, at def

기본값은 정의 시점에 딱 한 번

원인은 단순해요. def 줄이 실행될 때(함수가 만들어질 때) 기본값 [] 가 한 번 평가돼 리스트 객체 하나가 생기고, 함수에 딱 붙어 보관돼요. 이후 items 없이 호출할 때마다 매개변수가 그 하나의 객체에 이름표로 붙죠 — 즉 모든 호출이 별칭! 그래서 제자리 변경(append)이 호출 사이에 그대로 남아요.

→ 기본 객체는 def 때 1회 생성·함수에 보관. 호출마다 그 하나에 이름표 붙임(별칭).

Step 02 — Fix It with None

None 관용구로 고친다

정석 해법: 기본값을 불변인 None으로 두고, 함수 안에서 새 리스트를 만들어요. def add(v, items=None): if items is None: items = []. 함수 몸통은 호출할 때마다 실행되니, items = [] 도 매번 실행돼 매 호출 새 리스트가 생기죠. 토글로 함정 버전과 고친 버전을 비교하세요 — 같은 세 번 호출인데 결과가 갈려요.

→ 기본값은 None(불변), 새 객체는 함수 안에서. 그럼 매 호출 새 리스트.

Step 03 — Takeaways

정리

기본값은 def 실행 시점에 한 번 평가돼 함수에 보관돼요. 가변 기본값([]·{})은 모든 호출이 공유하는 별칭이 되어 내용이 쌓이죠. 고치려면 기본값을 None으로 두고 함수 안에서 새로 만들기. 원인(별칭)도 해법(바인딩)도 PART B에서 배운 그대로예요. 여기까지 "이름표가 어떤 객체를 가리키나"였다면 — 이제 "그 이름표가 어디에 사는가"로 가요. 같은 이름도 사는 방이 다르거든요.
