PART B · 별칭과 복사 — CH 06

얕은 복사 vs 깊은 복사

별칭이 싫으면 복사하면 되죠. y = x.copy() 는 진짜 새 객체를 만들어 별칭을 끊어요. 하지만 함정이 있어요 — 이건 얕은 복사라서 겉(최상위)만 새로 만들고, 안에 든 객체는 원본과 공유해요. 그래서 중첩 리스트라면 안쪽을 바꿨을 때 여전히 같이 변하죠. 끝까지 독립시키려면 깊은 복사가 필요해요. "복사했는데 왜 같이 바뀌지?"의 답이 여기 있어요.

Step 00 — Copy Breaks the Alias

복사는 진짜 새 객체를 만든다

y = x 는 별칭(같은 객체)이었지만, y = x.copy() 는 새 리스트 객체를 만들어요(id가 다름). 이제 x와 y는 서로 다른 객체라, 평평한 리스트라면 한쪽을 바꿔도 다른 쪽은 그대로예요. 별칭이 끊긴 거죠. 토글로 별칭과 복사를 비교해 보세요 — 객체가 하나냐 둘이냐가 갈려요.

→ copy()는 새 객체(id 다름) → 별칭 끊김. 단, "겉"만 새로 만들어요(다음 단계).

Step 01 — Shallow Copy Shares the Inside

겉만 새 것, 속은 공유

함정은 중첩(리스트 안의 리스트)에서 드러나요. x = [[1, 2], 3] 을 얕은 복사하면, 바깥 리스트는 새로 만들지만 안쪽 리스트 [1, 2] 는 원본과 같은 객체를 가리켜요(주소만 복사). 그래서 y[0].append(9) 로 안쪽을 바꾸면 — x의 안쪽도 같이 변해요! 겉은 독립인데 속은 여전히 별칭인 거죠.

→ 얕은 복사: 최상위만 새 객체, 안의 객체는 주소만 복사(공유). 중첩이면 여전히 위험.

Step 02 — Deep Copy Goes All the Way

깊은 복사: 끝까지 새로 만든다

깊은 복사(copy.deepcopy(x))는 바깥부터 안쪽까지 모든 객체를 재귀적으로 새로 만들어요. 그래서 안쪽 리스트도 완전히 별개 — 어디를 바꿔도 원본은 전혀 안 변하죠. 완전한 독립 사본이에요. 토글로 얕은/깊은 복사에서 안쪽을 바꿨을 때 원본이 어떻게 다른지 비교하세요.

→ 깊은 복사 = 끝까지 재귀 복사 → 완전 독립. 얕은 복사는 한 겹만.

Step 03 — Takeaways

정리

얕은 복사(.copy())는 최상위만 새로 만들고 안의 객체는 공유 — 평평한 리스트엔 충분하지만 중첩이면 안쪽이 여전히 별칭이에요. 깊은 복사(deepcopy)는 끝까지 복사해 완전 독립. "복사했는데 같이 바뀐다"면 십중팔구 중첩 + 얕은 복사예요. 여기까지가 별칭·복사 함정. 이제 또 다른 악명 높은 함정 — 함수의 기본값이 호출마다 새로 생기지 않는다는 사실로 가요.
