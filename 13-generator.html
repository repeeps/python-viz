PART D · 게으른 흐름 — CH 13

제너레이터

값을 미리 다 만들어 리스트에 쌓아두는 대신, 필요할 때 하나씩 만들어 내보내는 함수 — 그게 제너레이터예요. return 대신 yield 를 쓰면, 함수는 값을 하나 내주고 바로 그 자리에서 멈춰요(상태를 그대로 간직한 채). 다음 값을 요청하면 멈췄던 곳에서 재개하죠. 제너레이터는 이터레이터의 한 종류라 1회용이지만, 덕분에 무한 수열도 메모리 걱정 없이 다룰 수 있어요.

Step 00 — yield Pauses

return은 끝, yield는 멈춤

보통 함수는 return 하면 끝나고 모든 게 사라져요. 제너레이터는 yield 로 값을 하나 내주고 그 자리에서 일시정지해요 — 지역 변수와 진행 위치를 그대로 간직한 채로요. 죽지 않고 "잠든" 상태죠. 다음 값을 달라고 하면 거기서 깨어나 이어가요. def 안에 yield 가 하나라도 있으면 그 함수는 제너레이터가 돼요.

→ yield = 값 하나 내주고 상태를 간직한 채 멈춤(잠듦). return처럼 끝나지 않아요.

Step 01 — Pause and Resume

멈췄던 곳에서 이어간다

count() 가 n 을 yield하고 멈춰요. next() 로 다음 값을 요청하면 — 멈췄던 yield 다음 줄부터 깨어나 n += 1 을 하고 다시 yield에서 멈추죠. 지역 변수 n 이 호출 사이에 기억되는 게 핵심이에요(보통 함수면 매번 초기화될 텐데). 버튼으로 멈춤·재개를 따라가 보세요.

→ 재개는 멈춘 다음 줄부터. 지역 상태(n)가 호출 사이에 보존돼요.

Step 02 — Infinite, Tiny Memory

무한 수열도 메모리 걱정 없이

"모든 짝수"처럼 끝없는 수열을 리스트로 만들면? 불가능해요 — 메모리가 터지죠. 제너레이터는 값을 미리 안 만들고 요청할 때 하나씩만 내주니, 무한 수열도 문제없어요. 지금 필요한 하나만 메모리에 있거든요. 토글로 비교하세요 — 리스트는 전부를 미리 담지만(즉시·전부), 제너레이터는 그때그때(게으르게·하나씩).

→ 리스트=전부 미리 만들어 메모리에 / 제너레이터=요청 시 하나씩(무한도 OK).

Step 03 — Takeaways

정리

제너레이터는 yield 로 값을 하나 내주고 상태를 간직한 채 멈췄다, 다음 요청에 재개하는 함수예요. 이터레이터의 한 종류라 1회용이지만, 값을 미리 다 안 만들어 무한 수열·거대 데이터를 작은 메모리로 다룰 수 있죠(게으른 평가). 여기까지 — 우리는 함수를 변수에 담고, 돌려주고(클로저), yield로 멈추는 특별한 함수까지 봤어요. 이 모든 게 가능한 건 파이썬에서 함수도 객체이기 때문이에요. 마지막 PART에서 그 의미를 봐요.
