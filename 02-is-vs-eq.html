PART A · 이름과 객체 — CH 02

같은 객체? 닮은 객체?

두 가지 "같다"가 있어요. == 는 "내용이 같나?"(동등성), is 는 "같은 객체인가?"(동일성)예요. 값이 똑같은 두 리스트는 == 는 참이지만 is 는 거짓 — 닮았지만 별개의 객체거든요. 그래서 값 비교엔 항상 == 를, is 는 None 같은 "단 하나뿐인" 객체 확인에만 써요. (==의 행동은 객체가 __eq__로 직접 정해요.)

Step 00 — Two Kinds of Equal

내용이 같다 vs 같은 객체다

a = [1, 2] 와 b = [1, 2] — 똑같이 생긴 리스트 두 개를 따로 만들었어요. 둘은 내용이 같으니 a == b 는 참. 하지만 서로 다른 객체(다른 id)라서 a is b 는 거짓이에요. 닮은 쌍둥이지 같은 사람이 아니죠. id를 보면 둘이 다른 객체임이 보여요.

→ == 는 내용 비교, is 는 객체 동일성. 값이 같아도 객체는 다를 수 있어요.

Step 01 — Same Object

이름표 둘이 한 객체를 가리키면

이번엔 b = a — CH01에서 봤듯 새 객체가 아니라 같은 객체에 이름표 추가예요. 그럼 둘은 같은 객체(같은 id)니 a is b 도 참, 당연히 a == b 도 참. 토글로 두 상황(따로 만듦 vs 같은 객체)을 나란히 비교해 보세요 — is 의 결과가 갈려요.

→ is 는 "같은 객체?"를 정확히 짚어요. 별칭(다음 PART) 판별의 도구.

Step 02 — None, and a Trap

None은 단 하나 · is의 함정

None 은 프로그램 전체에 딱 하나만 존재하는 특별한 객체예요(싱글톤). 그래서 "값이 None인가?"는 x is None 으로 확인하는 게 정석이죠. 반대로 숫자·문자열을 is 로 비교하면 안 돼요. 파이썬이 작은 정수 같은 걸 몰래 재사용하기도 해서 is 가 우연히 맞아 보일 때가 있거든요 — 토글로 그 변덕을 보세요. 규칙: 값 비교는 언제나 ==.

→ is 는 None류(싱글톤)에만. 숫자/문자열 값 비교엔 절대 쓰지 말 것 — 캐싱 범위는 외우지 마세요.

Step 03 — Takeaways

정리

== 는 내용(동등성), is 는 같은 객체인가(동일성). 값 비교엔 항상 ==, is 는 None 같은 싱글톤 확인에만. 그리고 ==가 어떻게 "내용이 같다"를 판단하는지는 객체가 __eq__로 직접 정해요(나중에 덕타이핑에서 다시 만나요). 이제 객체를 구분할 줄 알게 됐으니 — 어떤 객체는 내용이 바뀌고 어떤 건 안 바뀐다는 결정적 차이로 가요. 다음 챕터.
