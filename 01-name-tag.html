PART A · 이름과 객체 — CH 01

이름표 붙이기

파이썬에서 대입(=)은 "상자에 값을 담는" 게 아니라 "객체에 이름표를 붙이는" 일이에요. x = [1, 2] 를 쓰면 리스트 객체가 메모리에 생기고, x는 거기 붙은 이름표일 뿐이죠. 그래서 한 객체에 이름표를 여러 개 붙일 수도 있고(y = x), del로 이름표를 뗄 수도 있어요. 이 단순한 그림이 앞으로 나올 모든 함정의 열쇠예요.

Step 00 — Assignment Is Tagging

대입은 이름표를 붙인다

한 줄을 단계로 따라가 보세요. x = [1, 2] 가 실행되면 ① 먼저 오른쪽의 리스트 객체가 메모리에 만들어지고, ② 그다음 왼쪽의 이름 x가 그 객체에 이름표로 붙어요. 순서가 중요해요 — 객체가 먼저, 이름은 나중. 이름은 객체를 "담는" 그릇이 아니라 그냥 가리키는 표예요.

→ 오른쪽 객체 생성 → 왼쪽 이름표 붙이기. 이름은 객체를 가리킬 뿐.

Step 01 — Many Tags, One Object

한 객체에 이름표 여럿

y = x 는 새 객체를 만들지 않아요. x가 가리키던 바로 그 객체에 이름표 y를 하나 더 붙일 뿐이죠. z = y 도 마찬가지. 그래서 x·y·z 셋 다 같은 리스트를 가리켜요. 버튼으로 이름표가 늘어나는 걸 보세요 — 객체는 계속 하나뿐이에요. (이게 다음 챕터 "별칭"의 씨앗이에요.)

→ y = x 는 복사가 아니라 이름표 추가. 객체는 하나, 이름표는 여럿.

Step 02 — del Removes a Tag

del은 객체가 아니라 이름표를 뗀다

초보자가 흔히 오해하는 것 — del x 는 객체를 지우는 게 아니에요. 이름표 x만 떼는 거죠. 다른 이름표(y)가 아직 붙어 있으면 객체는 멀쩡히 살아 있어요. 마지막 이름표마저 떼서 아무도 안 가리키게 되면, 그제서야 파이썬이 객체를 청소해요. 버튼으로 따라가 보세요.

→ del = 이름표 떼기. 마지막 이름표가 사라진 객체만 청소됨.

Step 03 — Takeaways

정리

대입은 객체에 이름표 붙이기, del은 이름표 떼기예요. 한 객체에 이름표가 여럿일 수 있고, 이름표가 0개가 되면 객체는 청소돼요. 그런데 — 이름표 두 개가 "같은 객체"를 가리키는 것과, 두 객체가 "값이 같은" 것은 다른 얘기죠? 그걸 어떻게 구분할까요? 다음 챕터.
