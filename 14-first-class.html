PART E · 모든 게 객체 — CH 14

함수도 값이다

파이썬에선 모든 게 객체예요 — 숫자도, 리스트도, 그리고 함수도. 함수 이름은 그저 함수 객체에 붙은 이름표일 뿐이죠(CH01 그대로). 그래서 함수를 다른 변수에 담고(say = greet), 인자로 넘기고, 리스트에 넣고, 함수에서 돌려받을 수 있어요. 이걸 "함수가 일급(first-class)"이라고 해요. PART C에서 클로저로 함수를 돌려받았던 것도 다 이 덕분이었어요.

Step 00 — Functions Are Objects

함수 이름도 그냥 이름표

def greet(): ... 를 정의하면, 함수 객체가 하나 만들어지고 이름표 greet 가 거기 붙어요 — 리스트나 숫자와 똑같아요. 그래서 say = greet 를 하면(괄호 없이!) 같은 함수 객체에 이름표 say 가 하나 더 붙죠. 이제 say() 도 greet() 와 똑같이 동작해요. 괄호 () 는 "호출", 괄호 없는 이름은 "함수 객체 자체"예요.

→ 함수도 객체, 이름은 이름표. greet(괄호X)=객체 자체, greet()=호출.

Step 01 — Passing Functions Around

함수를 인자로 넘긴다

함수가 값이니, 다른 함수에 인자로 넘길 수 있어요. apply_twice(f, x) 는 받은 함수 f 를 x 에 두 번 적용하죠. apply_twice(double, 3) 이면 double 함수가 들어가 3→6→12. sorted(…, key=함수)·map(함수, …) 같은 게 다 이 원리예요 — "무엇을 할지"를 함수로 건네는 거죠. 버튼으로 함수가 인자로 들어가 실행되는 걸 보세요.

→ 함수를 인자로 넘겨 "무엇을 할지"를 건넴(map·sorted key 등). 고차 함수.

Step 02 — Returning Functions

함수를 돌려받는다

함수는 다른 함수를 만들어 돌려줄 수도 있어요 — CH11 클로저에서 본 그대로죠. make_multiplier(3) 이 "3배 하는 함수"를 만들어 돌려주면, 받은 쪽은 그걸 변수에 담아 마음껏 호출해요. 함수가 값이라 가능한 일이에요. 이 "함수를 받아서 / 함수를 돌려주는" 조합이 — 다음 챕터 데코레이터의 정확한 재료예요.

→ 함수가 함수를 돌려줌(클로저). 받기+돌려주기 = 데코레이터의 재료.

Step 03 — Takeaways

정리

파이썬에서 함수도 객체라, 이름표로 가리키고 변수에 담고 인자로 넘기고 돌려받을 수 있어요(일급 함수). 이게 클로저·고차 함수를 가능케 했죠. 그리고 "함수를 받아서 살짝 바꾼 함수를 돌려주는" 패턴이 아주 유용해서, 파이썬은 그걸 위한 깔끔한 문법까지 줬어요 — @. 그게 데코레이터예요. 다음 챕터.
