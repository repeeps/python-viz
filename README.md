# 파이썬, 눈으로 보기

파이썬을 쓸 때 **머릿속/메모리에서 실제로 무슨 일이 일어나는지** — 이름표가 어떤 객체를 가리키고, 무엇이 바뀌고 안 바뀌는지 — 를 수식·코드 강좌 없이 **움직이는 그림과 인터랙션**으로 익히는 비영리 학습 자료입니다. 0번 입문 + 16챕터.

> 문법을 가르치지 않아요. 대신 **멘탈 모델**을 바꿔 끼워요 — "변수는 값을 담는 상자"가 아니라 "객체에 붙이는 이름표". 이 한 장면이 별칭 버그·가변 기본 인자·루프 클로저 같은 파이썬의 악명 높은 함정을 전부 같은 그림으로 풀어줘요.

## 어떻게 보나

별도 빌드·설치가 없습니다. `index.html`을 브라우저로 열면 됩니다. 모든 이름표↔객체 그림·id 변화·스코프 탐색은 브라우저에서 실시간으로 그려집니다.

## 구조

처음이라면 **0번부터 순서대로**. 척추는 **이름과 객체(A) → 별칭과 복사(B) → 스코프(C) → 게으른 흐름(D) → 모든 게 객체(E)**.

- **0번 — 변수는 상자가 아니다**: 상자 모델을 깨고 이름표 모델로 · 용어(객체·이름·참조·가변/불변) · 코스맵
- **PART A · 이름과 객체**
  - 01 이름표 붙이기 (대입=바인딩 · del=이름표 떼기)
  - 02 같은 객체? 닮은 객체? (is vs == · None 싱글톤 · 정수 캐싱 함정)
  - 03 가변 vs 불변 (제자리 변경 vs 새 객체 · += 두 얼굴)
- **PART B · 별칭과 복사** — 함정의 진원지 (기둥①)
  - 04 재바인딩 vs 제자리 변경 (id로 구분)
  - 05 별칭 — 한 객체, 여러 이름 (함수 인자도 바인딩 · "값/참조 전달" 반박)
  - 06 얕은 복사 vs 깊은 복사
  - 07 가변 기본 인자 함정 (def 시점 1회 평가 · None 관용구)
- **PART C · 이름이 사는 곳 — 스코프** (기둥②)
  - 08 네임스페이스 (이름→객체 방)
  - 09 LEGB — 이름 찾는 순서 (가림)
  - 10 대입이 만드는 지역변수 (UnboundLocalError · global/nonlocal)
  - 11 클로저 (늦은 바인딩 · 루프 클로저 함정)
- **PART D · 게으른 흐름**
  - 12 이터러블 ≠ 이터레이터 (소진)
  - 13 제너레이터 (yield 멈춤/재개 · 무한·메모리)
- **PART E · 모든 게 객체**
  - 14 함수도 값이다 (일급 함수)
  - 15 데코레이터 (클로저 + 일급 함수의 합작)
  - 16 덕타이핑 + 회고 (능력으로 판단 · 전체 멘탈 모델 한 장)

두 기둥은 **별칭과 복사(B)** 와 **스코프(C)** — 초보자가 가장 자주 빠지는 함정이 다 거기서 나와요.

## 기술

- 순수 정적 HTML + JavaScript (의존성·빌드 없음)
- 공통 엔진 `shared/lib.js`(VZ 코어 + 스텝 플레이어 VZ.AL + heatColor) + `shared/lib-py.js`(파이썬 도메인 엔진 VZ.PY: 이름표·객체 상자(가변/불변)·참조 화살표·네임스페이스·LEGB 스코프층·이터레이터 커서·제너레이터 상태 프레임·데코레이터 감싸기) — 모두 순수 함수(상태 → SVG)
- 다크 테마 `shared/theme.css`
